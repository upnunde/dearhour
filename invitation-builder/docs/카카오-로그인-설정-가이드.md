# 카카오 로그인 설정 가이드 (디자이너·기획용)

개발 코드는 이미 연결되어 있습니다. **카카오 개발자 사이트**와 **Supabase 웹 관리 화면**에서 값만 맞추면 로그인이 동작합니다. 아래 순서대로 진행하세요.

---

## 왜 두 군데를 설정하나요?

| 곳 | 하는 일 (비유) |
| --- | --- |
| **카카오** | “우리 앱으로 로그인해 주세요” 창을 띄우고, 로그인이 끝나면 **Supabase**에게 결과를 넘깁니다. |
| **Supabase** | 그 결과를 받아서 회원 세션을 만들고, 다시 **우리 웹사이트**(예: dearhour)로 보냅니다. |

그래서  
- 카카오에는 “로그인 끝나면 **Supabase 주소**로 보내라”고 적고,  
- Supabase에는 “카카오가 나를 믿을 수 있게 **앱 비밀번호(Client Secret)** 를 알려 달라”고 적습니다.

---

## 미리 알아두면 좋은 말

- **Redirect URI (리다이렉트 URI)**  
  로그인이 끝난 뒤 **브라우저가 이동할 정확한 주소**입니다. 오타·`http`와 `https` 혼동만으로도 실패합니다.

- **REST API 키**  
  카카오 앱마다 부여되는 **긴 문자열**입니다. Supabase 화면에서는 보통 **Client ID** 칸에 넣습니다.

- **Client Secret**  
  카카오에서 “보안” 옵션을 켜면 나오는 **비밀 코드**입니다. Supabase의 **Client Secret** 칸에 넣습니다. (메모장에 안전하게 보관하세요.)

- **프로젝트 ref (레퍼런스)**  
  Supabase 주소 `https://여기부분.supabase.co` 에서 **여기부분** 한 덩어리입니다.  
  예: 주소가 `https://abcdxyz123.supabase.co` 이면 ref는 `abcdxyz123` 입니다.

---

## 0단계: 메모장 하나 열기

아래에서 복사한 값들을 메모장에 순서대로 적어 두면 실수가 줄어듭니다.

1. Supabase 콜백 주소 (한 줄 전체)  
2. REST API 키  
3. Client Secret (있을 경우)

---

## 1단계: Supabase에서 “콜백 주소” 한 줄 만들기

1. 브라우저에서 [Supabase 대시보드](https://supabase.com/dashboard)에 로그인합니다.  
2. **이 서비스에 쓰는 프로젝트**를 선택합니다.  
3. 브라우저 주소창이나 프로젝트 설정에서 **Project URL** 을 확인합니다.  
   - 형태: `https://(영문·숫자).supabase.co`  
4. 괄호 안의 덩어리를 **프로젝트 ref** 라고 부릅니다.  
5. 메모장에 아래처럼 **한 줄**로 적습니다. (ref만 본인 것으로 바꾸세요.)

```text
https://(프로젝트ref).supabase.co/auth/v1/callback
```

**예시** (ref가 `abcdxyz123` 인 경우):

```text
https://abcdxyz123.supabase.co/auth/v1/callback
```

이 줄 전체가 카카오에 넣을 **Redirect URI** 입니다.  
`/auth/v1/callback` 까지 포함되어야 하고, **앞에 다른 글자를 붙이면 안 됩니다.**

---

## 2단계: 카카오 개발자 콘솔

1. [Kakao Developers](https://developers.kakao.com) 에 카카오 계정으로 로그인합니다.  
2. 상단 메뉴 **내 애플리케이션** 을 누릅니다.  
3. 앱이 없으면 **애플리케이션 추가하기** 로 앱을 만듭니다. (앱 이름 등은 안내대로 입력)

### 2-1. 카카오 로그인 켜기

1. 왼쪽 또는 상단에서 **제품 설정** → **카카오 로그인** 으로 갑니다.  
2. **사용 설정** 이 **ON** 인지 확인합니다. OFF면 켭니다.

### 2-2. Redirect URI 등록 (가장 중요)

1. 같은 **카카오 로그인** 설정 영역 안에서 **Redirect URI** 를 찾습니다.  
2. **URI 추가** 또는 **등록** 버튼을 누릅니다.  
3. **1단계**에서 메모장에 적어 둔 주소 **전체**를 복사해서 붙여넣고 저장합니다.

   - `https://` 로 시작하는지  
   - 끝이 `/auth/v1/callback` 인지  
   한 번 더 확인하세요.

### 2-3. REST API 키 복사 (Supabase의 Client ID)

1. 왼쪽 **앱 키** 메뉴(또는 앱 요약 화면의 키 표)로 갑니다.  
2. **REST API 키** 옆의 값을 **전체 복사**해 메모장에 적습니다.  
   - 이 값을 Supabase의 **Client ID** 칸에 넣습니다.

### 2-4. Client Secret (Supabase가 비밀번호를 요구할 때)

Supabase Kakao 설정에 **Client Secret** 칸이 있고, 비워두면 안 되는 경우:

1. **카카오 로그인** 관련 메뉴에서 **보안** 또는 **Client Secret** (이름은 카카오 UI 업데이트에 따라 조금 다를 수 있음)을 찾습니다.  
2. **Client Secret 코드 사용** 을 활성화하고, 생성된 **코드**를 복사합니다.  
3. 화면에 “한 번만 보여줍니다” 같은 안내가 있으면 **반드시 그때 메모장에 저장**합니다.

Secret을 나중에 잃어버리면 **재발급**하면 되고, 재발급 후에는 Supabase에도 **새 값**을 다시 저장해야 합니다.

---

## 3단계: Supabase에 카카오 값 입력

1. [Supabase 대시보드](https://supabase.com/dashboard) → 해당 **프로젝트** 선택  
2. 왼쪽 메뉴 **Authentication** (인증) 클릭  
3. 상단 또는 목록에서 **Providers** (제공자)  
4. 목록에서 **Kakao** 찾기  
5. **Enable** (또는 Kakao 사용) 스위치를 **켭니다**  
6.  
   - **Client ID**: 메모장의 **REST API 키** 붙여넣기  
   - **Client Secret**: 메모장의 **Client Secret** 붙여넣기 (없으면 카카오에서 먼저 2-4 진행)  
7. **Save** 저장

---

## 4단계: “우리 사이트 주소”도 Supabase에 허용되게

로그인 마지막에 사용자가 **우리 도메인**으로 돌아와야 합니다. Supabase가 그 주소를 허용 목록에 넣어 두어야 합니다.

1. Supabase → **Authentication** → **URL Configuration**  
2. **Redirect URLs** (리다이렉트 URL 목록)을 봅니다.  
3. 아래 형태가 **포함**되어 있는지 확인하고, 없으면 **추가**합니다.

실제 서비스 주소 예 (본인 도메인으로 바꾸세요):

```text
https://dearhour.co.kr/auth/callback
```

`www` 를 쓰는 주소도 쓰면 **둘 다** 넣는 것이 안전합니다.

```text
https://www.dearhour.co.kr/auth/callback
```

로컬에서 테스트할 때만 (개발자 PC):

```text
http://localhost:3000/auth/callback
```

포트 번호(`3000` 등)는 **실제로 브라우저 주소창에 보이는 것**과 같아야 합니다.

4. **Site URL** 이 실제 서비스 메인 주소와 맞는지도 같이 확인합니다.

---

## 5단계: 실제로 눌러 보기

1. 배포된 사이트(또는 테스트 URL)의 **로그인** 페이지로 갑니다.  
2. **카카오로 계속하기** 버튼을 누릅니다.  
3. 카카오 동의 화면이 뜨고, 끝나면 **마이페이지 등**으로 돌아오면 성공입니다.

설정을 저장한 직후에는 반영에 **잠깐** 걸릴 수 있으니, 한두 번 실패하면 1분 정도 뒤에 다시 시도해 보세요.

---

## 잘 안 될 때 (증상별로 볼 곳)

| 증상 | 먼저 확인할 것 |
| --- | --- |
| 카카오 화면에서 “redirect_uri …” 오류 | 카카오 Redirect URI에 넣은 주소가 **1단계 한 줄**과 **완전히 동일**한지 (대소문자·슬래시 포함) |
| “invalid client” 류 | Supabase의 Client ID가 **REST API 키 전체**인지, Secret은 **최신 발급값**인지 |
| 카카오는 됐는데 우리 사이트에서 오류 문구 | Supabase **Redirect URLs**에 `https://본인도메인/auth/callback` 이 있는지 |
| 로컬만 안 됨 | 주소가 `localhost:3000` 인지 `3001` 인지 확인하고, Redirect URLs에 **보이는 주소 그대로** 추가 |

개발자에게 전달할 때는 **스크린샷**(카카오 Redirect URI 화면, Supabase Kakao Provider 화면, URL Configuration 화면)을 찍어 주시면 원인 파악이 빨라집니다. **REST API 키·Secret 값 자체**는 채팅/메일에 그대로 올리지 마세요.

## 한 페이지 요약 체크리스트

- [ ] 카카오: 카카오 로그인 **ON**  
- [ ] 카카오: Redirect URI = `https://(ref).supabase.co/auth/v1/callback`  
- [ ] 카카오: REST API 키 복사해 둠  
- [ ] 카카오: Client Secret 필요 시 발급·복사  
- [ ] Supabase: Providers → Kakao **ON**, Client ID / Secret 저장  
- [ ] Supabase: Redirect URLs에 `https://본인도메인/auth/callback` (및 www·로컬 필요 시)  
- [ ] 사이트에서 **카카오로 계속하기** 테스트
